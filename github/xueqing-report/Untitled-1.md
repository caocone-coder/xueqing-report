
学情报告迭代

# 核心任务

针对学习机学情报告模块迭代，核心目标：提升有效报告查看率，同时牵引不同分层家长使用自研学情应用。

# 现存核心痛点

1. 报告可读性差：日/周报内容冗长、多屏滑动，信息过载，用户抓不住重点，完整查看率低

2. 计划学情脱节：学习计划与学情分析分散无联动，用户无法依据实际学情快速调整计划

3. 无分享功能：无法便捷分享孩子学习成果，满足不了家长展示进步的需求

4. 学科数据碎片化：各学科数据独立展示，需反复切换页面才能查看完整学情

5. 自研应用价值传递弱：无清晰介绍和场景引导，家长不了解优势与使用场景，认知模糊

# 用户分层现状

- 整体以活跃用户为主，三方活跃用户为最大群体

- 学情活跃用户占比偏低，存在部分沉睡、流失用户

- 数据差异：仅学情活跃用户报告数据完整，其余分层用户报告数据偏少

# 迭代核心要求

针对不同用户分层定制差异化学情报告，依托报告引导家长了解、使用学习机自研学情应用。
